# Svelte on CE

A Vite + Svelte + TypeScript starter: a counter that persists to the CE database (`ce.db.set`, a mesh-replicated map) and broadcasts changes over a live mesh room (`ce.room('counter')`), so every open tab and device stays in sync. Run `npm i && npm run build` to produce a static `dist/` with relative asset paths (it serves correctly under `/apps/<id>/` or on a custom subdomain), then `ce-app serve` to run it locally or `ce-app expose` to publish it over the mesh. The local `src/ce.ts` is the mesh-native `@ce/client`: db and rooms ride gossipsub via your local node only (the in-browser `window.__ceNode` bridge or the same-origin `/ce` proxy), never a remote origin, so a strict same-origin CSP holds. Swap it for the published `@ce/client` package if you prefer.
