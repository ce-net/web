# React Native (Expo) on CE

A single React Native UI built with Expo, exported to the web via react-native-web: `npm i && npm run build` runs `expo export --platform web --output-dir dist` to produce a static web bundle served by your local CE node (`baseUrl: ""` keeps asset paths relative for `/apps/<id>/`). On the web target the counter persists to the CE database through `ce.db` (a mesh-replicated map); the same component also runs natively under `expo start` on iOS and Android. Run it locally with `ce-app serve` or publish over the mesh with `ce-app expose`. The local `ce.ts` is the mesh-native `@ce/client`: on the web build it talks only to your local node (the in-browser `window.__ceNode` bridge or the same-origin `/ce` proxy), never a remote origin, so a strict same-origin CSP holds. Swap it for the published `@ce/client` package if you prefer. Expo pulls a large dependency tree, so the first `npm i` is heavier than the Vite templates.
