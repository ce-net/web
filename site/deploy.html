Three commands. You're live.

Node 18+ is the only prerequisite. The first command scaffolds a chat app; swap in any template name.

1

Scaffold an app (chat by default)

$ npm create ce-app
2

Step into it

$ cd my-ce-app
3

Build, watch, and push live to the mesh — prints your public URL

$ npx ce-app dev

You get a public dev URL, instantly.

Every app gets a stable id the first time you run it — saved to .ce/app-id. Your build is uploaded to the hub and served from a real URL anyone can open. Hot reload is automatic: a tiny injected snippet watches your app's version and reloads the page the instant you save.

https://ce-net.com/apps/<your-id>/

That's your live app — multi-file static sites and SPAs alike. Share the link; it works on any phone, any laptop, anywhere. Run npx ce-app deploy for a one-shot upload without watching.

A database and realtime, in two sentences.

Both ship in the @ce/client ESM lib, namespaced to your app automatically. No setup, no keys.

🗄️

CE database

A persistent, namespaced key–value store — read, write, delete, and list keys newest-first for histories and feeds. Survives restarts.

const { db } = createClient(); await db.set('msg:42', { text: 'hi' }); const recent = await db.list('msg:', 50);
⚡

Realtime rooms

Pub/sub over a websocket: join a room, anything you send is broadcast to everyone else in it, with the last 50 messages replayed on connect.

const r = createClient().room('lobby'); r.on(msg => render(msg)); r.send({ text: 'hello mesh' });
