Run work on the mesh

WASM · instant

Each button ships a tiny WebAssembly workload to a live node and brings back the answer — with the node that ran it and how long it took. Tap one.

Write a task, run it on a node

JavaScript · sandboxed

Write a function. Hit run. It's shipped to a live node, executed in an isolated worker over there, and the return value comes back. This is the whole point of CE — your code, someone else's hardware.

Content-addressed storage

blobs · verifiable

Type something (or drop a file). It's hashed on this device, stored on the mesh under that hash, then you can pull it back from anywhere by the hash — and we re-verify the bytes match. Tamper-proof by construction.

Run a language model — on this device

WebGPU

A small instruct model, downloaded once and run entirely in this browser tab via WebGPU. No server, no API key — the tokens below are generated by your node. First load fetches the weights (~300 MB, cached after).

Run a node without a browser

one line

The tab is great for a quick join. For an always-on node, run the headless worker — same protocol, no browser, works on macOS / Linux / Windows. Then push work to it from anywhere.

Headless node — needs Node 22+ (zero dependencies)curl -fsSL https://ce-net.com/worker.js | node - --hub wss://ce-net.com/hub
Push a task from any machine — lands on a live nodecurl -sX POST https://ce-net.com/hub/tasks -H 'content-type: application/json' -d '{"func":"count_primes","args":[200000]}'
Push your own code — runs in a sandbox on a nodecurl -sX POST https://ce-net.com/hub/tasks -H 'content-type: application/json' -d '{"lang":"js","code":"i=>i.a+i.b","input":{"a":2,"b":40}}'
