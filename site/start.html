01

Run code on the mesh

live · sandboxed

Write a function. Hit Run. Your code is shipped to a live node somewhere on the mesh, executed in an isolated worker over there, and the return value comes back — with the node that ran it and how long it took. This is the whole idea: your code, someone else's hardware.

Each snippet ships {lang:"js",code,input} to POST /hub/tasks. The prime-count example uses the prebuilt WASM demo module instead. No nodes online? You'll get a friendly note — open a browser node in another tab and try again.

02

Deploy an app in one command

hot-reload

Scaffold, run, and get a live public URL in seconds. ce-app dev watches your files and hot-reloads a hosted dev preview — share the link, edit locally, it updates instantly.

Scaffold a new app (pick a template when prompted)$ npm create ce-app@latest myapp
Enter it$ cd myapp
Run with a live, hot-reloading URL$ npx ce-app dev

Prints a live URL like https://<project>-<your-id>.ce-net.com/ that hot-reloads as you save. When you're ready: npx ce-app deploy to publish, ce-app slug claim <name> to grab a clean ce-net.com/apps/<name>/ URL, and ce-app publish to list it in the public registry.

Realtime rooms

websocket · multiplayer

Open a websocket to /rt/<app>/<room> and any text frame you send is broadcast to everyone else in the room (the last 50 frames replay on connect). That's the whole realtime API. You're connected to start-demo/lobby below — open this page in a second tab and watch them talk.

05

Languages & SDKs

polyglot

Compute tasks run three ways: as WebAssembly, as sandboxed JavaScript, or as Python (Pyodide CPython on browser nodes). Client SDKs let you push work, read data, and join rooms from anywhere.

runtime

WASM

Ship a compiled module + a function name. Deterministic, fast, language-agnostic. Used by the prime demo above.

runtime

JavaScript

Send a function as source; it runs in an isolated worker with a hard wall-clock cap. The default for ad-hoc tasks.

runtime

Python

Pyodide-backed CPython on browser nodes. Define task(input), return anything JSON-serializable.

TypeScript

@ce/client

The browser/Node SDK: tasks, db, and realtime rooms in a few lines. Powers ce-app.

Python

Python client

Push tasks and read the KV store from scripts and notebooks over the same HTTP API.

Go · Rust

Go & ce-rs

A Go client and the Rust SDK (ce-rs) speak the node HTTP API directly — money as base units, no floats.

// @ce/client — push a task, read/write data, join a room
import { tasks, db, room } from "@ce/client";

const out = await tasks.run({ lang: "js", code: "i => i.a + i.b", input: { a: 2, b: 40 } });
await db.put("myapp", "score", { value: 42 });
const r = room("myapp", "lobby");
r.on("message", (m) => console.log(m));
r.send({ hello: "mesh" });

06

Build a distributed multiplayer game

authoritative

The netgame framework handles the hard parts of real multiplayer on the mesh: authoritative host election (one node simulates the world), input relay, state broadcast, and failover from a /db snapshot if the host node drops — another node picks up the simulation mid-flight. You write the game; the mesh keeps it running. See Co-op Arena for a full server-authoritative ship arena built this way.

// the minimal shape — the framework does election, relay, snapshot & failover
createGame({
  init() { return { ships: {}, scores: {} }; },        // initial world
  tick(state, dt) { /* advance physics on the host */ },
  input(state, playerId, msg) { /* apply a client's input */ },
  onState(state) { /* render the latest world on every client */ },
});
